# 主题颜色配置指南

## 🎨 快速更换强调色

本网站已实现全局主题颜色系统，只需修改一个文件即可更换整个网站的强调色。

### 📁 配置文件位置
```
/config/theme.ts
```

### 🎯 如何更换强调色

1. **打开配置文件**
   ```bash
   vim config/theme.ts
   ```

2. **修改强调色配置**
   ```typescript
   export const themeColors = {
     primary: {
       DEFAULT: '#DC143C',        // 修改这里 - 主要强调色
       light: '#FF1744',          // 修改这里 - 浅色变体
       glow: '#FF0000',           // 修改这里 - 光晕色
       bright: '#FF5252',         // 修改这里 - 明亮色变体
     },
     // ... 其他配置保持不变
   };
   ```

3. **保存并重新构建**
   ```bash
   npm run build
   ```

### 🔧 技术实现

#### 颜色命名规范
- `accent` - 主要强调色
- `accent-light` - 浅色变体
- `accent-glow` - 光晕效果
- `accent-bright` - 明亮色变体

#### 自动应用位置
- ✅ 按钮背景和边框
- ✅ 链接悬停效果
- ✅ 图标颜色
- ✅ 渐变背景
- ✅ 阴影效果
- ✅ 表单焦点状态
- ✅ 文本选择样式
- ✅ 滚动条样式

#### CSS 类名映射
```css
.bg-accent              /* 主要强调色背景 */
.text-accent            /* 主要强调色文字 */
.border-accent          /* 主要强调色边框 */
.bg-gradient-accent     /* 强调色渐变背景 */
.hover:text-accent      /* 悬停时文字变色 */
.shadow-glow           /* 强调色光晕阴影 */
```

### 📝 注意事项

1. **颜色和谐性**: 建议选择与暗黑主题搭配良好的颜色
2. **对比度**: 确保文字在深色背景下有足够的对比度
3. **品牌一致性**: 选择符合品牌调性的颜色
4. **可访问性**: 考虑色盲用户的可访问性

### 🚀 高级自定义

如需更深度的主题定制，可以：

1. **修改其他颜色组**
   ```typescript
   background: {},      // 背景色组
   text: {},           // 文字色组
   metal: {},          // 金属色组
   card: {},           // 卡片色组
   status: {},         // 状态色组
   ```

2. **自定义渐变**
   ```typescript
   gradients: {
     accent: `linear-gradient(135deg, ${主色} 0%, ${浅色} 100%)`,
   }
   ```

3. **自定义阴影**
   ```typescript
   shadows: {
     glow: `0 0 20px ${主色}40`,
   }
   ```

### 💡 最佳实践

1. **测试所有状态**: 确保悬停、焦点、激活状态都正常显示
2. **响应式测试**: 在不同设备上验证颜色效果
3. **构建验证**: 修改后务必运行 `npm run build` 确认无错误
4. **浏览器测试**: 在不同浏览器中测试颜色渲染

### 📞 支持

如需帮助或有疑问，请：
1. 检查 `npm run build` 是否有错误
2. 查看浏览器控制台是否有 CSS 警告
3. 确认颜色值格式正确（十六进制 #RRGGBB）
