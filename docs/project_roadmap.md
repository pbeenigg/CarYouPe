# CarYouPe 项目开发路线图 (Project Roadmap)

本路线图旨在有序推进项目开发。根据最新的开发策略，我们将**优先完成管理后台与后端的全部功能**，然后再进行小程序端的开发，以确保后端 API 的稳定性和数据管理的完备性。

## 📅 Phase 1: 管理后台与后端核心 (Admin & Backend First)

**目标**: 完成管理后台所有功能及对应后端接口，确保数据管理闭环。

### 1.1 基础设施 (Infrastructure) ✅

- [x] **后端基础**: 数据库迁移 (Alembic)，认证模块 (JWT)，API 路由框架。
- [x] **后台基础**: Next.js 项目初始化，Shadcn UI 配置，i18n 多语言，登录页面对接。

### 1.2 用户与权限管理 (User Management)

- [x] **用户 API**: 后端 `/admin/users` 增删改查接口。
- [x] **用户管理页**: 前端用户列表展示、状态管理。
- [ ] **角色权限**: 管理员/代理商/普通用户的权限隔离逻辑。

### 1.3 商品与订单中心 (Product & Order)

- [x] **商品 API**: 后端 `/admin/products` 接口，支持 SKU 和车型匹配。
- [x] **商品管理页**: 前端商品列表、上架/下架操作。
- [ ] **商品发布**: 复杂表单开发（图片上传、规格编辑器、适用车型选择）。
- [ ] **订单管理**: 后端订单模型与接口，前端订单列表及发货流程。

### 1.4 分销与财务系统 (Distribution & Finance)

- [ ] **分销逻辑**: 后端关系链绑定、佣金计算引擎。
- [ ] **钱包系统**: 后端余额/积分/提现接口。
- [ ] **财务后台**: 前端分销商审核、提现审批、佣金记录查询页面。

---

## 📅 Phase 2: 小程序端开发 (UniApp Development)

**目标**: 基于已完善的后端 API，开发用户端小程序，实现业务闭环。

### 2.1 框架搭建 (Framework)

- [ ] **项目初始化**: UniApp (Vue3 + TS) 项目搭建。
- [ ] **UI 集成**: 配置 uView Plus 组件库。
- [ ] **导航配置**: 实现 TabBar (首页、分类、购物车、我的)。

### 2.2 核心交易流程 (Core Commerce)

- [ ] **首页装修**: 轮播图、金刚区、推荐流。
- [ ] **分类与搜索**: 品牌->车系->车型三级联动筛选。
- [ ] **交易闭环**: 商品详情 -> 购物车 -> 结算页 -> 模拟支付。

### 2.3 分销中心 (Distribution Center)

- [ ] **个人中心**: 用户信息展示、订单状态概览。
- [ ] **分销推广**: “我的团队”、“佣金明细”、“推广海报”页面。

---

## 📅 Phase 3: 多端适配与特色功能 (Advanced Features)

**目标**: 提升用户体验，覆盖更多流量平台。

### 3.1 AI 预览 (AI Preview)

- [ ] **AI 对接**: 集成图像生成服务。
- [ ] **实景渲染**: 实现脚垫效果图在不同车型内饰中的实时预览。

### 3.2 多端适配 (Cross-Platform)

- [ ] **平台兼容**: 调试并修复 微信、抖音、小红书 三端的登录与支付差异。

---

## 📅 Phase 4: 测试与部署 (Testing & Deployment)

**目标**: 系统稳定上线。

### 4.1 全链路测试

- [ ] **单元测试**: 核心业务逻辑测试。
- [ ] **压力测试**: 高并发场景模拟。

### 4.2 生产部署

- [ ] **容器化**: Dockerfile 编写与编排。
- [ ] **CI/CD**: 自动化构建与发布流程。
