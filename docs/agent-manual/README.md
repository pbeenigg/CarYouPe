## 什么是 Agent 配置？

Agent 配置是一套增强 AI 助手能力的配置系统，包括：

### 1. Agents (专业代理)

预定义的专业角色，每个 Agent 有特定的专长领域，如架构设计、代码审查、测试驱动开发等。

### 2. Commands (斜杠命令)

使用 `/command` 格式触发的快捷操作，如 `/plan`、`/tdd`、`/code-review` 等。

### 3. Contexts (上下文模式)

定义 AI 助手的工作模式和行为偏好，如开发模式、研究模式、审查模式。

### 4. Skills (技能库)

可重用的知识和最佳实践集合，AI 助手可以在需要时参考这些技能。

## 快速开始

### 使用命令

在对话中输入斜杠命令即可触发相应功能：

```
/plan 我需要添加用户认证功能
/tdd 实现一个计算市场流动性的函数
/code-review
```

### 选择上下文

根据当前工作类型选择合适的上下文：

- 开发新功能时 → **dev** 模式
- 研究问题时 → **research** 模式
- 审查代码时 → **review** 模式

## 最佳实践

1. **先规划后执行** - 使用 `/plan` 命令制定实施计划
2. **测试驱动开发** - 使用 `/tdd` 确保代码质量
3. **定期代码审查** - 使用 `/code-review` 检查代码问题
4. **参考技能库** - 遇到特定场景时查阅相关 Skill

## 自定义配置

您可以根据项目需求自定义配置：

- 创建新的 Agent：在 `.agent/agents/` 目录添加 `.md` 文件
- 创建新的命令：在 `.agent/commands/` 目录添加 `.md` 文件
- 创建新的技能：在 `.agent/skills/` 目录添加子目录和 `SKILL.md`

详细配置格式请参考各子目录的使用手册。
