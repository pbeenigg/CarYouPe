# Commands 使用手册

## 概述

Commands（斜杠命令）是快捷触发特定工作流程的方式。在对话中输入 `/command-name` 即可激活相应功能。

## 命令配置格式

每个命令配置文件使用 YAML frontmatter：

```markdown
---
description: 命令的简短描述
---

# 命令标题

命令详细说明...
```

## 可用命令列表

### 规划类命令

#### /plan - 制定实施计划

**描述**：重述需求、评估风险、创建分步实施计划

**特点**：

- 在编写任何代码之前等待用户确认
- 创建详细的阶段性计划
- 识别依赖和风险

**使用场景**：

- 开始新功能
- 重大架构变更
- 复杂重构工作
- 需求不明确时

**示例**：

```
/plan 我需要添加实时通知功能当市场结算时
```

**输出格式**：

```
# 实施计划: 实时市场结算通知

## 需求重述
- 当用户关注的市场结算时发送通知
- 支持多种通知渠道（应用内、邮件、webhook）
- 确保通知可靠送达

## 实施阶段

### 阶段 1: 数据库架构
...

### 阶段 2: 通知服务
...

**等待确认**: 是否按此计划执行？(是/否/修改)
```

---

#### /orchestrate - 协调多代理工作

**描述**：协调多个 Agent 完成复杂任务

### 开发类命令

#### /tdd - 测试驱动开发

**描述**：强制执行 TDD 工作流，先生成测试再编写代码

**TDD 流程**：

1. 定义接口/类型
2. 编写失败测试 (RED)
3. 实现最少代码 (GREEN)
4. 重构改进 (REFACTOR)
5. 验证覆盖率 (80%+)

**使用场景**：

- 实现新功能
- 添加新函数/组件
- 修复 bug（先写重现测试）
- 重构现有代码

**示例**：

```
/tdd 我需要一个计算市场流动性评分的函数
```

---

#### /build-fix - 修复构建错误

**描述**：诊断并修复构建错误

**使用场景**：

- `npm run build` 失败
- TypeScript 编译错误
- 依赖问题

**示例**：

```
/build-fix
```

---

#### /refactor-clean - 重构清理

**描述**：重构和清理代码

**使用场景**：

- 代码需要重构
- 清理技术债务
- 改善代码结构

---

### 测试类命令

#### /e2e - 端到端测试

**描述**：编写和运行 E2E 测试

**使用场景**：

- 验证用户流程
- 完成功能后验证
- 回归测试

**示例**：

```
/e2e 测试用户注册流程
```

---

#### /test-coverage - 测试覆盖率

**描述**：检查和改进测试覆盖率

**目标覆盖率**：

- 常规代码：80%+
- 关键业务逻辑：100%

---

#### /verify - 验证检查

**描述**：运行验证循环（构建、类型检查、lint、测试）

**检查项目**：

1. Build 验证
2. Type 检查
3. Lint 检查
4. 测试套件
5. 安全扫描
6. Diff 审查

**输出格式**：

```
验证报告
==================

Build:     [通过/失败]
Types:     [通过/失败] (X 个错误)
Lint:      [通过/失败] (X 个警告)
Tests:     [通过/失败] (X/Y 通过, Z% 覆盖率)
Security:  [通过/失败] (X 个问题)
Diff:      [X 个文件变更]

总体:      [就绪/未就绪] 可以提 PR
```

---

### 审查类命令

#### /code-review - 代码审查

**描述**：检查代码质量、安全性和可维护性

**检查优先级**：

1. **关键**：安全漏洞、硬编码密钥
2. **高**：大函数、缺少错误处理
3. **中**：性能问题、代码风格
4. **低**：建议改进

## 命令组合使用

### 典型开发流程

```
1. /plan 添加用户认证功能     # 先规划
2. [用户确认计划]
3. /tdd 实现登录功能          # TDD 开发
4. /build-fix                 # 如果构建失败
5. /code-review               # 代码审查
6. /verify                    # 验证检查
7. /update-docs               # 更新文档
```

### 修复 Bug 流程

```
1. /plan 修复 [bug 描述]      # 分析问题
2. /tdd 编写重现测试          # 先写测试
3. /test-coverage             # 确保覆盖
4. /code-review               # 审查修复
```

## 自定义命令

创建新命令的步骤：

1. 在 `.agent/commands/` 目录创建 `.md` 文件
2. 添加 YAML frontmatter 定义描述
3. 编写命令说明和使用指南

**模板**：

```markdown
---
description: 简短描述命令功能
---

# 命令名称

## 功能说明

此命令用于...

## 使用场景

- 场景 1
- 场景 2

## 使用示例

\`\`\`
/command-name 参数
\`\`\`

## 相关命令

- /related-command-1
- /related-command-2
```
